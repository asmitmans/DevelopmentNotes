
# 03.7 — Implementación de JwtService

En este capítulo incorporamos JWT al proyecto,
pero aún no lo conectamos al endpoint login.

---

- JwtService creado.
- Métodos para:
  - Generar token
  - Extraer username
  - Validar token
- La aplicación compila correctamente.

---

# Requisitos Previos

Se asume completado:

03.6 — Login Endpoint

- Autenticación con username/password funcionando.

---

# 1. Dependencias JJWT

Se agregan las dependencias modernas:

- jjwt-api
- jjwt-impl
- jjwt-jackson

Versión utilizada:

0.13.0

La API se incluye en compile.
La implementación y Jackson se incluyen en runtime.

---

# 2. Configuración del Secret

En application.properties:

security.jwt.secret-base64=...
security.jwt.ttl-seconds=3600
security.jwt.issuer=SpringSecurityJWTDemo

El secret debe:

- Ser generado de forma segura.
- Estar codificado en Base64.
- Tener tamaño adecuado para HS256.

---

# 3. Responsabilidad de JwtService

JwtService debe ser responsable de:

1. Generar token firmado.
2. Extraer claims.
3. Validar firma y expiración.

No debe:

- Acceder a base de datos.
- Interactuar con SecurityContextHolder.
- Conocer HttpRequest.

Debe ser un servicio puro de token.

---

# 4. Generación del Token

Al generar el token:

Se establecen los siguientes claims:

- subject (username)
- issuer
- issuedAt
- expiration

Se firma con HS256 utilizando la SecretKey.

El resultado es un String compacto.

---

# 5. Validación del Token

Para validar:

1. Verificar firma.
2. Extraer subject.
3. Verificar expiración.
4. Comparar username con UserDetails.

Si algo falla:

- Se considera inválido.

---

# 6. Flujo Conceptual

Login exitoso →
JwtService.generateToken(userDetails) →
Token firmado

Request posterior →
JwtService.extractUsername(token) →
JwtService.isTokenValid(token, userDetails)

---
