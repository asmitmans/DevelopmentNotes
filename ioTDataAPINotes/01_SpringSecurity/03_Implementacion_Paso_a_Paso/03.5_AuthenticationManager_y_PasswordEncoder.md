# 1. PasswordEncoder

Spring Security no compara passwords en texto plano.

El flujo correcto es:

passwordIngresado → encoder.matches(passwordIngresado, passwordEnBD)

Por eso definimos un bean:

PasswordEncoder

Usamos:

BCryptPasswordEncoder

Ventajas:

- Hash adaptativo
- Seguro para producción
- Estándar de facto en Spring

Las contraseñas en la base deben almacenarse ya encriptadas.

Nunca se almacena texto plano.

---

# 2. AuthenticationManager

AuthenticationManager es el componente que:

- Recibe credenciales
- Delegará al AuthenticationProvider
- Validará usuario y contraseña
- Lanzará excepción si falla

Por defecto, Spring lo crea internamente.

En aplicaciones modernas, debemos exponerlo como bean:

authenticationManager(AuthenticationConfiguration config)

Esto nos permite inyectarlo en nuestro controlador
para construir un endpoint login personalizado.

# 3. Flujo Completo hasta Ahora
