# 03.2 — Modelo de Dominio y Configuración JPA

En este capítulo definimos el modelo de datos básico
para soportar autenticación de usuarios.

---

# Estado esperado al finalizar este capítulo

- Entidad User creada.
- Entidad Role creada.
- Relación Many-to-Many configurada.
- Repositorios JPA definidos.
- H2 configurado en memoria.
- Consola H2 accesible.
- Datos persistidos correctamente.

---

# Requisitos Previos

Se asume completado:

03.1 — Proyecto Base

El proyecto debe levantar correctamente.

---

# 1. Entidad User

Representa el usuario autenticable del sistema.

Responsabilidades:

- Contener username
- Contener password
- Contener roles
- Ser entidad persistente

Relación:

User
  └── ManyToMany
        Role

El username debe ser único.

La contraseña se almacenará en formato encriptado
(BCrypt más adelante).

---

# 2. Entidad Role

Representa un permiso o autoridad del sistema.

Ejemplos:

- ROLE_USER
- ROLE_ADMIN

Se mantiene separada de User para permitir
extensibilidad futura.

---

# 3. Relación entre User y Role

Relación Many-to-Many.

Un usuario puede tener múltiples roles.
Un rol puede pertenecer a múltiples usuarios.

Se utiliza una tabla intermedia:

user_roles

---

# 4. Repositorios JPA

Se crean:

- UserRepository
- RoleRepository

Responsabilidades:

UserRepository:
- Buscar por username

RoleRepository:
- Buscar por nombre de rol

---

# 5. Configuración H2 (application.properties)

Configuración típica:

- Base en memoria
- ddl-auto = update
- Consola H2 habilitada
- URL: /h2-console

Esto permite inspeccionar la base durante desarrollo.

---

# 6. Validación Manual

1. Levantar aplicación.
2. Acceder a:
   http://localhost:8080/h2-console
3. Verificar que se crean las tablas:
   - user
   - role
   - user_roles

Si aún no existen datos, es normal.
