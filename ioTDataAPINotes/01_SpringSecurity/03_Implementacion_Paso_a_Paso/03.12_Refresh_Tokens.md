# 03.12 — Refresh Tokens (opcional)

En este capítulo introducimos el concepto de refresh tokens.

Esto permite renovar tokens de acceso
sin requerir que el usuario vuelva a iniciar sesión.

---

# Estado esperado al finalizar este capítulo

- Access token de corta duración.
- Refresh token de mayor duración.
- Endpoint para renovar access token.

---

# Requisitos Previos

Se asume completado:

03.11 — Roles y autorización.

---

# 1. Problema con los access tokens

Los JWT normalmente tienen expiración corta.

Ejemplo:

15 minutos  
30 minutos  
1 hora

Esto mejora la seguridad,
pero obliga al usuario a autenticarse nuevamente
cuando el token expira.

---

# 2. Solución: Refresh Token

Se utilizan dos tokens.

Access Token

- duración corta
- usado en cada request

Refresh Token

- duración larga
- usado solo para obtener nuevo access token

---

# 3. Flujo típico

Login →

Servidor devuelve:

access_token  
refresh_token

Cuando access_token expira:

Cliente llama:

POST /auth/refresh

Servidor valida refresh token
y genera nuevo access token.

---

# 4. Ventajas

- Mejor experiencia de usuario
- Mayor seguridad
- Tokens de acceso cortos

---

# 5. Consideración de seguridad

Refresh tokens normalmente:

- se almacenan en base de datos
- pueden revocarse
- tienen mayor control que access tokens

---

# Objetivo de este Capítulo

Presentar el mecanismo de refresh tokens
