# 03.6 — Endpoint de Login (POST /auth/login)

En este capítulo implementamos un endpoint
de autenticación basado en username y password.

Aún no utilizamos JWT.

---

# Estado esperado al finalizar este capítulo

- Endpoint POST /auth/login implementado.
- Autenticación mediante AuthenticationManager.
- Login probado con curl.
- Respuesta simple en JSON.
- Sin generación de JWT todavía.

---

# Requisitos Previos

Se asume completado:

03.5 — AuthenticationManager y PasswordEncoder

- PasswordEncoder definido.
- AuthenticationManager expuesto como bean.
- UserDetailsService funcional.

---

# 1. Objetivo del Endpoint

Recibir:

{
  "username": "...",
  "password": "..."
}

Validar credenciales usando AuthenticationManager.

Si son correctas:

- Retornar respuesta HTTP 200.
- Devolver JSON simple.

Si son incorrectas:

- Spring lanzará AuthenticationException.
- Se devolverá 401 o 403 según configuración.

---

# 2. Flujo Interno

Request →
AuthController →
AuthenticationManager.authenticate(...) →
AuthenticationProvider →
UserDetailsService →
PasswordEncoder

Si todo es válido, el método no lanza excepción.

---

# 3. Estructura del Controlador

Se define:

- AuthController
- DTO LoginRequest
- (Respuesta simple en JSON)

Importante:

El endpoint debe estar permitido en SecurityConfig:

.requestMatchers("/auth/login").permitAll()

Y CSRF debe ignorarlo:

.csrf().ignoringRequestMatchers("/auth/login")

---

# 4. Pruebas Manuales

## Login correcto

```bash
curl -i -X POST http://localhost:8080/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"user","password":"1234"}'
````

Resultado esperado:

HTTP/1.1 200

Respuesta JSON simple (ejemplo):

{
"message": "Login successful"
}

---

## Login incorrecto

```bash
curl -i -X POST http://localhost:8080/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"user","password":"mala"}'
```

Resultado esperado:

HTTP/1.1 401 (o 403 según entrypoint)
