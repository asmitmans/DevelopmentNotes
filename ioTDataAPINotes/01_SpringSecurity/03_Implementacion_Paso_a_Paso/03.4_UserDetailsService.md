# 1. ¿Qué es UserDetailsService?

Es la interfaz que Spring Security utiliza
para cargar usuarios durante autenticación.

Responsabilidad:

Recibir un username y devolver un UserDetails.

Spring nunca consulta directamente tu entidad User.
Siempre lo hace a través de esta interfaz.

---

# 2. Implementación personalizada

Creamos una clase:

MyUserDetailsService

Responsabilidades:

- Inyectar UserRepository.
- Buscar usuario por username.
- Lanzar excepción si no existe.
- Convertir entidad User en UserDetails.

La conversión incluye:

- Username
- Password
- Authorities (roles convertidos a GrantedAuthority)

---

# 3. Conversión de Roles a Authorities

Spring Security trabaja con GrantedAuthority.

Por eso:

Role → SimpleGrantedAuthority

Ejemplo conceptual:

ROLE_USER → new SimpleGrantedAuthority("ROLE_USER")

Es importante mantener el prefijo ROLE_.

---

# 4. Qué cambia en la aplicación

Antes:

Spring usaba un usuario generado automáticamente.

Ahora:

Spring usará los usuarios almacenados en H2.

Esto permite controlar autenticación real.

---

# 5. Validación Manual

Si aún no existe un endpoint login personalizado,
Spring puede seguir mostrando login form.

Sin embargo:

Internamente ya estará usando nuestro UserDetailsService.

Podemos verificar:

- Insertar un usuario manualmente en H2.
- Intentar login (cuando implementemos endpoint).
