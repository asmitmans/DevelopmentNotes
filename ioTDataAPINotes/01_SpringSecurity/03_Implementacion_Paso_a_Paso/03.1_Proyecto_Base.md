# 1. Generación del Proyecto

Usar Spring Initializr con:

- Project: Maven
- Language: Java
- Spring Boot: 4.0.3
- Packaging: Jar
- Java: 21 (o la versión definida en el proyecto)

- Group: tu “dominio” invertido (identidad de organización).
    - Genérico: com.tuempresa / cl. tunombre / io.tuproyecto
    - Regla: estable, en minúsculas, sin espacios.
- Artifact: el nombre técnico del módulo (carpeta + nombre del .jar).
    - Genérico: (ejemplo) spring-security-jwt-demo o springsecurity-jwt-demo
    - Regla: corto, sin espacios, normalmente en kebab-case
- Name: nombre “humano” del proyecto (lo verás en el IDE).
    - Genérico: SpringSecurityJWTDemo
    - Regla: puede ser más bonito, no afecta tanto como artifact
- Description: texto libre (solo documentación/metadata).
    - Genérico: “Demo project for Spring Security using JWT”.
- Package name: paquete base donde quedará tu clase Application y desde donde
  Spring hace component scan.
    - Regla típica: Group + "." + Artifact (pero en formato de paquete)
    - Ejemplo: si Group=cl.agustin y Artifact=springsecurityjwtdemo →
      cl.agustin.springsecurityjwtdemo
- Packaging:
    - Jar (recomendado casi siempre): app Spring Boot ejecutable (java -jar).
    - War: solo si vas a desplegar en un servidor externo tipo Tomcat tradicional.

Ejemplo:
- Group: cl.tunombre
- Artifact: springsecurity-jwt-demo
- Name: SpringSecurityJWTDemo
- Package name: `cl.tunombre.springsecurity

Dependencias:

- Spring Web
- Spring Data JPA
- H2 Database

# 4. Endpoint de Prueba

## Crear un controlador mínimo:

En el package `controller` creamos un controlador para las pruebas
`PingController`

**Codigo de ejemplo:**
```java

import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/api/ping")
public class PingController {

    @GetMapping
    public ResponseEntity<String> getPong() {
        return ResponseEntity.ok("pong\n");
    }
}
```

Como agregamos Spring Security a las dependencias,
por defecto todas las rutas quedan protegidas.Por 
lo que necesitaremos crear una configuracion de 
seguridad para las pruebas.

En el package `config` creamos la clase `SecurityConfig` donde 
autorizamos el acceso al recurso de prueba.

**Codigo de ejemplo:**
```java

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.web.SecurityFilterChain;

@Configuration
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
                .authorizeHttpRequests(auth -> auth
                        .requestMatchers("/api/ping").permitAll()
                        .anyRequest().authenticated()
                );
        return http.build();
    }
}
```

## Prueba 

GET /api/ping

Debe devolver:

pong

**Probar:**
```bash
curl http://localhost:8080/api/ping
```
Resultado esperado:
```bash
pong
```
