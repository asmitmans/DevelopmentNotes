# 03.8 — Login devuelve JWT

En este capítulo conectamos el JwtService
al endpoint de autenticación.

El login ahora devuelve un token firmado.

Aún no procesamos el token en requests posteriores.

---

# Estado esperado al finalizar este capítulo

- POST /auth/login genera JWT.
- Respuesta incluye:
  {
    "token": "...",
    "type": "Bearer"
  }
- Login probado con curl.
- Aún no existe JwtFilter.
- Endpoints protegidos siguen devolviendo 401/403.

---

# Requisitos Previos

Se asume completado:

03.7 — JwtService

- Servicio JWT implementado.
- Secret configurado.
- Token generable y validable.

---

# 1. Modificación del AuthController

El flujo ahora es:

1. Recibir username y password.
2. Autenticar con AuthenticationManager.
3. Cargar UserDetails.
4. Generar token con JwtService.
5. Devolver JSON con token.

No se almacena sesión.
No se usa SecurityContext manualmente.

---

# 2. Estructura de Respuesta

Se crea DTO:

LoginResponse

Campos:

- token
- type (Bearer)

Esto facilita consumo por frontend o cliente REST.

---

# 3. Flujo Interno

Request →
AuthenticationManager.authenticate(...) →
UserDetailsService →
JwtService.generateToken(...) →
Response JSON

---

# 4. Pruebas Manuales

## Login correcto

```bash
curl -i -X POST http://localhost:8080/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"user","password":"1234"}'
````

Resultado esperado:

HTTP/1.1 200

{
"token": "<JWT>",
"type": "Bearer"
}

---

## Login incorrecto

```bash
curl -i -X POST http://localhost:8080/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"user","password":"mala"}'
```

Resultado esperado:

HTTP/1.1 401 (o 403 según entrypoint)

---

# 5. Prueba de Endpoint Protegido (aún sin filtro)

Aunque tengamos token:

```bash
curl -i http://localhost:8080/api/ping \
  -H "Authorization: Bearer <JWT>"
```

Resultado esperado:

401 o 403

Esto es correcto.

El backend todavía no procesa el token.
